How to Deploy a Plain HTML Site Using GitHub Pages

GitHub Pages is an excellent way to host static websites for free — no servers, no fuss, just push your files and your site is live!

In this post, I’ll walk you through how to deploy a plain HTML page on GitHub Pages, ideal for simple portfolios, project pages, or blogs without a CMS.

Why GitHub Pages?

  • Free hosting directly from your GitHub repo
  • Supports custom domains
  • Automatically publishes your site on every push
  • Supports Jekyll for static site generation (optional)
  • Great for tech blogs, documentation, portfolios, and project demos

Step 1: Create Your GitHub Repository

First, create a new repo on GitHub:

  1. Go to https://github.com/new
  2. Name your repo (e.g., my-html-site)
  3. Optionally add a README (you can edit later)
  4. Click Create repository

Step 2: Clone Your Repo Locally

Open your terminal and clone the repo to your machine:

git clone https://github.com/yourusername/my-html-site.git
cd my-html-site

Step 3: Add Your HTML File

Create an index.html file in your repo folder. This will be the homepage of your site.

Example index.html content:

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <title>My Simple HTML Site</title>
</head>
<body>
  <h1>Welcome to My Static Site!</h1>
  <p>This site is deployed using GitHub Pages.</p>
</body>
</html>

Save the file inside your repo directory.

Step 4: Commit and Push Your Changes

Add your new file and push it to GitHub:

git add index.html
git commit -m "Add homepage"
git push origin main

Step 5: Enable GitHub Pages

  1. On GitHub, go to your repository’s Settings tab
  2. Select Pages from the sidebar
  3. Under Source, select the branch (main) and the folder (/root or /docs if you placed your site files there)
  4. Click Save

GitHub will build and deploy your site in a few minutes.

Step 6: Visit Your Live Site

Your site will be available at:

https://yourusername.github.io/my-html-site/

If you named your homepage index.html, it will load automatically.

Tips for Managing Your GitHub Pages Site

  • Use folders for assets: Store images, CSS, and JS files in folders like /assets or /static to keep your repo organized.
  • Relative paths: Use relative links for your assets to avoid broken references.
  • Custom domain: You can add a custom domain in the Pages settings and include a CNAME file in your repo.
  • Jekyll integration: If you want dynamic content, you can enable Jekyll by adding _config.yml and markdown files.
  • Ignore files: Use .gitignore and _config.yml exclude/include rules to manage what gets deployed.

Common Pitfalls to Avoid

  • Not naming your homepage index.html: Browsers look for index.html by default.
  • Using absolute URLs for assets: This can break your site if deployed under a subpath.
  • Placing site files in wrong folder: Make sure you select the right source folder for GitHub Pages in settings.
  • Ignoring cache: GitHub Pages may cache your site for a few minutes after updates.

Conclusion

Deploying a static HTML site with GitHub Pages is straightforward, reliable, and free. Whether you want to showcase your projects, write a simple blog, or share documentation, GitHub Pages makes it easy to get your content online with minimal fuss.
