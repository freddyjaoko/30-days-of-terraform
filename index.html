Infrastructure Deployments

Single Server Deployment

  • Region: us-east-1
  • Instance Type: t2.micro
  • Key Features: Security group allowing ingress on port 22 (SSH)

This deployment helped me understand the basics of provisioning a simple server in AWS using Terraform, including managing security groups and instance configuration.

Web Server Deployment

  • Region: us-east-1
  • Instance Type: t2.micro
  • Key Features: Security group allowing ingress on port 80 (HTTP)

This allowed me to practice setting up a publicly accessible web server, making sure port 80 was open to receive web traffic.

Challenges and Solutions

One challenge I encountered was managing AWS key pairs. Initially, I struggled with Terraform failing to locate or properly use my SSH key. To fix this, I manually imported my public key into AWS through the console, then referenced the existing key pair in my Terraform configuration instead of trying to create one automatically.

This taught me the importance of managing external resources carefully and how to work around Terraform’s limitations with AWS key management.
